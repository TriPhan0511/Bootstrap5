Bootstrap Containers

Containers are a fundamental buliding block of Bootstrap that contain, pad, and align your content within a given device or viewport.

How they work

Containers are the most basic layout element in Bootstrap and are required when using our default grid system. Contains are used to contain, pad, and (sometimes) center the content within them. While containers can be nested, most layouts do not require a nested container.

Bootstrap comes with three different containers:

  • .container, which sets a max-width at each responsive breakpoint.
  • .container-fluid, which is width: 100% at all breakpoints.
  • .container-{breakpoint}, which is width: 100% until the specifed breakpoint.

Default container

Our default .container class is a responsive, fixed-width container, meaning its max-width changes at each breakpoint.

Responsive container

Responsive containes allow you to specify a class that is 100% wide until the specified breakpoint is reached, after which we apply max-widths for each higher breakpoint. For example, .container-sm is 100% wide to start until the sm breakpoint is reached, where it will scale up with md, lg, xl, and xxl.

Fluid container

Use .container-fluid class for a full width container, spanning the entire width of the viewport.
